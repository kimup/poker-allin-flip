# Open Questions

現時点では実装を止めるほどの不明点はありません。最初は仮決めで進められます。

## 決めたいこと

1. 公開先はVercelで進めてよいか

   基本方針はVercelです。GitHub Pagesとの比較は [Hosting Comparison](hosting-comparison.md) に整理します。

2. 初期UIは日本語だけでよいか

   まずは日本語でよさそうです。将来英語切り替えを入れる場合は文言を分離します。

3. 1人モードの「運がいい」の基準をどうするか

   候補:

   - 単純な勝敗数
   - 実勝率
   - プリフロップ勝率との差分
   - オールインEVとの差分

   初期版は勝敗数と実勝率から始め、次に期待勝率との差分を入れるのが現実的です。

4. ハンド指定の優先度

   MVP後すぐ入れるなら、まずは `AA`、`AKs`、`QQ` のような簡易指定から始めるのがよさそうです。具体カード指定はその次で十分です。

5. 見た目の方向性

   候補:

   - ポーカーテーブル風
   - シンプルな分析ツール風
   - スマホゲーム風

   2人で同じ画面を見る用途があるので、カードと勝敗が大きく見える方向が合いそうです。

## 仮決め

- 技術: Vite + React + TypeScript
- 最初は静的サイト
- 役判定はライブラリ利用
- データ保存はlocalStorageまで
- `main` 公開、`develop` 作業
- 最初の機能はランダムハンド同士のフリップ
